Description: This website provides a list of all checkpoints from WCAG guidelines. They are separated by priority based on the item's impact of user accessibility. For each checkpoint, indicate if the item is present.

Priority 1
Text Alternatives

Text descriptions should be available for images and other visual items. All non-text content should have a text alternative that serves an equilvalent purpose.

Descriptive Labels

Descriptive labels for all user interface components such as buttons/other interactive elements

Multimodal Touch Screen Gestures

Allow multiple forms of input (to accomodate those with dexterity issues can't pinch so have alternative for them)

Context-Sensitive Help

Some users with disabilities may be more likely to make mistakes than users without disabilities. Using context-sensitive help, users find out how to perform an operation without losing track of what they are doing.

Ease of Seeing Content

Color should be used to convey information, indicate an action, prompt a response, or distinguish a visual element. The colors used on the application should use a contrast ratio of 3:1. Text cues should be provided for information displayed by color differences.

Ease of Hearing Content

Any audio should be controlled by starting, pausing, rewinding, and forwarding it. A mechanism should be provided in order to control audio volume independently from the device volume level

Intuitive Navigation

The navigation in the application starts in a predictable location without skipping over content.
