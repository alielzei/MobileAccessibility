MaximMobile: A Mobile Accessibilty Checklist

Instructions: This website provides a list of all checkpoints from WCAG guidelines. They are separated by priority based on the item's impact of user accessibility. For each checkpoint, indicate whether the item is present, not present, or not applicable.

Text descriptions should be available for images and other visual items. All non-text content should have a text alternative that serves an equilvalent purpose.

Equivalent Accessible Alternatives

If a user interface component is incompatible with an accessibility service, the application provides an alternative user interface component that complies with said accessibility service.

Indicative Guides

If the user has completed an action and needs to be guided to an additional action, there is an indicator that guides the user to complete the proper action. Ensure that there is audio/visual alternatives for accomodating screen readers.

Multiple Ways to Access Content

Make it possible for users to locate content in a manner that best meets their needs. Users may find one technique easier or more comprehensible to use than another.

Priority 3
Logical Tab Order

Elements need to follow a logical order: left to right, and from top to bottom.

Consistent Navigation Labeling

Navigational mechanisms that are present on more than one of the app interfaces are presented in the same relative order.

Replacements for Device Manipulation Gestures

Certain control operations can be performed by physically manipulating the device. These physical controls can pose challenges. Therefore, provide users with touch and keyboard operable controls.

Descriptions of Accessibility Features

An accessibility feature that is present within a mobile application should be described in the application’s documentation or user interface.

Changes Between Versions

If changes to any features are made to the application, especially concerning accessibility features, these changes are documented and presented to the user.

Display Corresponding Virtual Keyboard Per Data Entry Type

Customize the device settings for applicable devices in order to display the corresponding keyboard based on the data entry that is required. For example, numerical data entries should display the numerical/symbolic keyboard on the device.

Customizable User Interface Settings

Allow the user to have a mechanism to change the user interface settings, confirm the new changes, and reverse setting changes when desired. The user should also be able to restore the user interface settings to the default selections.
